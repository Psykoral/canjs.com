  • page
 
  • source

[10/16/13 10:36:20 AM] David Luecke: @page migrating Migrating to 2.0

With version 2.0 introducing new features like can.Component

Use .extend for extending Constructs

can.Observe and can.Observe.List becomes can.Map and can.List

In order to comply with the ECMAScript 6 terminology can.Observe has been renamed to can.Map and can.Observe.List is now called can.List. Additionally can.Observe.startBatch and can.Observe.stopBatch are now available as can.batch.start and can.batch.stop. Backwards compatible mappings to can.Observe and can.Observe.List are in place but should be considered deprecated.

can.route

It is now necessary to always call can.route.ready(true) once all routes have been set up. If you previously didn't use can.route.ready(true) specifically just add it on document ready.
